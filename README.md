# Classification-of-requests-by-topic
## 📋 Описание проекта

Модель машинного обучения для автоматической классификации обращений пользователей по тематикам на основе текста сообщения. Проект предназначен для обработки запросов в службу поддержки и позволяет автоматически распределять обращения по категориям.

## 🎯 Задача классификации

Модель классифицирует обращения по 4 типам:

- **Incident** - Инциденты (сбои, отказы систем)
- **Request** - Запросы (помощь, доступ, информация)
- **Problem** - Проблемы (ошибки, баги, некорректная работа)
- **Change** - Изменения (обновления, модификации)

## 🏗️ Архитектура модели

Модель построена с использованием TensorFlow/Keras и включает следующие слои:
Embedding(max_words=6000, output_dim=100)

SpatialDropout1D(0.3)

Conv1D(128, 3, padding='same', activation='relu')

MaxPooling1D(pool_size=2)

Conv1D(64, 3, padding='same', activation='relu')

MaxPooling1D(pool_size=2)

Bidirectional(LSTM(32, dropout=0.2))

BatchNormalization()

Dense(64, activation='relu')

Dropout(0.4)

Dense(32, activation='relu')

Dropout(0.3)

Dense(4, activation='softmax')

text

### Ключевые особенности архитектуры:

- **Conv1D слои** - быстрое извлечение локальных признаков
- **Bidirectional LSTM** - учет контекста в обоих направлениях
- **Dropout** - регуляризация для борьбы с переобучением
- **BatchNormalization** - стабилизация обучения
- **SpatialDropout1D** - улучшенная регуляризация для embedding слоя
